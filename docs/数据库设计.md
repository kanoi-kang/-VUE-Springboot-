# 新疆文旅电子商城 - 数据库设计文档

---

## 一、数据库概述

**数据库名称**：`xjwl_mall`  
**数据库类型**：MySQL 8.0+  
**字符集**：UTF-8  
**存储引擎**：InnoDB（支持事务和外键）

---

## 三、数据库关系图

```
用户模块                    商品模块                    订单模块
───────────────────────────────────────────────────────────────────
users 1 ─── * addresses     products 1 ─── * order_items          orders 1 ─── * order_items
   │                              │                    │                 │
   ├── * carts                   ├── categories (N:1)  └── refunds      ├── payments
   ├── * collections             ├── brands (N:1)                       └── refunds
   ├── * coupon_users            ├── * reviews
   └── * orders                  └── * product_specifications

优惠券模块
───────────
coupons 1 ─── * coupon_users

评价模块
───────────
reviews (关联 orders, products, users)
```

## 四、ER图关系说明

| 关系类型 | 关联表 | 说明 |
|---------|-------|-----|
| 1:N | users → addresses | 一个用户可有多条收货地址 |
| 1:N | users → carts | 一个用户可有多条购物车记录 |
| 1:N | users → collections | 一个用户可收藏多个商品 |
| 1:N | users → orders | 一个用户可创建多个订单 |
| 1:N | categories → products | 一个分类下有多个商品 |
| 1:N | brands → products | 一个品牌下有多个商品 |
| 1:N | products → product_specifications | 一个商品可有多个规格 |
| 1:N | orders → order_items | 一个订单包含多个订单项 |
| 1:N | orders → payments | 一个订单可有多次支付记录 |
| 1:N | orders → refunds | 一个订单可申请多次退款 |
| 1:N | coupons → coupon_users | 一张优惠券可被多个用户领取 |
| N:M | users ↔ coupons | 用户与优惠券多对多（通过coupon_users） |
| N:M | users ↔ products | 用户与商品多对多（通过collections） |

## 六、数据库初始化建议

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS xjwl_mall CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 使用数据库
USE xjwl_mall;

-- 设置时区
SET time_zone = '+08:00';
```

> **说明**：此数据库设计基于项目现有代码结构，涵盖用户管理、商品管理、订单管理、购物车、优惠券、评价等核心功能模块，支持完整的电商业务流程。
